1. Zoned Storage Devices

ZNS SSD는 장치의 LBA 공간을 고정 크기의 Zone으로 분할합니다. 각 Zone에는 쓰기 제약이 부여되며, 반드시 순차적으로만 쓸 수 있습니다.

2. Zone 동작 원리 (Write Pointer)

각 Zone은 Write Pointer (WP)를 유지합니다. WP는 해당 Zone 내에서 다음에 쓸 수 있는 LBA를 가리키는 커서입니다. 모든 쓰기는 반드시 WP가 가리키는 위치에서 시작해야 하며, 다른 오프셋에 쓰기를 시도하면 블록 레이어가 이를 거부합니다. 쓰기가 성공하면 WP는 쓴 블록 수만큼 자동으로 전진합니다.

Zone 0
FULL
Zone 1
OPEN
Zone 2
EMPTY
Zone 0: FULL (WP = Zone 끝) · Zone 1: OPEN (WP가 중간) · Zone 2: EMPTY (WP = Zone 시작)

Zone은 Zone Reset 명령을 발행한 후에만 재사용할 수 있으며, 이 명령은 WP를 Zone 시작 위치로 되돌리고 모든 데이터를 삭제합니다. 이러한 추가 전용(append-only) 모델은 NAND 플래시의 쓰기 전 삭제(erase-before-write) 특성을 그대로 반영하여, 복잡한 FTL 없이도 장치 컨트롤러가 플래시 페이지를 효율적으로 관리할 수 있게 합니다.

각 Zone이 순서대로만 쓰이고 원자적으로 초기화되기 때문에, 서로 다른 애플리케이션의 데이터가 같은 플래시 블록에 섞이지 않아 스트림 간 GC 간섭이 제거됩니다.

3. Zone 관리 명령

호스트는 아래 4가지 Zone 관리 명령으로 Zone의 상태를 명시적으로 제어합니다. Linux에서는 blkzone 유틸리티로 이 명령들을 발행합니다.

Zone Write Pointer 초기화
포인터를 Zone의 시작 위치로 이동합니다.
데이터가 삭제됩니다.
상태: Any → EMPTY
blkzone reset DEV
Zone 열기
앱이 지정한 Zone에 쓰기 연산을 수행할 것임을 장치에 명시적으로 알립니다.
장치는 해당 Zone에 내부 자원을 미리 할당합니다.
상태: EMPTY / CLOSED → EXPLICIT OPEN
blkzone open --offset --length DEV
Zone 닫기
다음 Open Zone 명령을 발행하기 전까지 앱이 이 Zone에 쓰기를 수행하지 않겠다고 선언합니다.
WP 위치는 유지되며 내부 자원이 해제됩니다.
상태: OPEN → CLOSED
blkzone close --offset --length DEV
Zone 완료
Write Pointer를 Zone의 끝으로 이동하여, 해당 Zone을 초기화하기 전까지 추가 쓰기를 차단합니다.
상태: Any → FULL
blkzone finish --offset --length DEV

4. Zone Append

순차 쓰기의 "Zone당 동시 I/O = 1" 제약을 극복하기 위해 NVMe ZNS Command Set은 Zone Append 명령을 정의합니다. 호스트는 정확한 LBA를 지정하지 않고 "이 Zone의 다음 위치에 데이터를 기록해 달라"는 방식으로 요청합니다. SSD 컨트롤러가 WP를 원자적으로 갱신하고, 실제 기록된 LBA를 완료 응답으로 반환합니다.
