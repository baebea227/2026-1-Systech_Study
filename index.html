실습 개요

본 실습은 ZNS(Zoned Namespace) SSD의 핵심 특성인 쓰기 제약, Zone 상태 전이, WAF(Write Amplification Factor)를 직접 측정하고 Conventional SSD와 비교·분석합니다.

QEMU + Linux 커널 환경에서 null_blk 모듈로 가상 ZNS 장치(/dev/nullb0)를 구성하고, fio·blkzone 명령으로 순차 쓰기 처리량 비교, Zone 상태 머신 조작, WAF 계산까지 Lab 1~3에서 단계별로 실험합니다. 이어지는 Lab 4 (Case Study)에서는 동일한 Zone 개념이 모바일 스마트폰(Zoned UFS)에 이식될 때 발생하는 시스템 과제와 해결 전략을 FAST '26 논문을 통해 탐구합니다.

1

배경 지식: ZNS SSD 구조와 쓰기 인터페이스

ZNS SSD의 등장 배경, Zone 구조와 Write Pointer, Zone 상태 머신(EMPTY → OPEN → FULL), Sequential Write 제약, Zone Append의 원자성과 LBA 반환, Active/Open Zone 제한, Bandwidth·Latency·Queue Depth(Little's Law)를 학습합니다.

개념 학습 Zone 상태 머신 Write Pointer Little's Law
2

실습 환경 준비: null_blk 기반 가상 ZNS 장치 구성

null_blk 커널 모듈을 소스에서 빌드·적재하고, configfs를 이용해 Zone 크기 256 MB / 전체 10 GB의 가상 ZNS 장치(/dev/nullb0)를 생성합니다. Zone 인식과 EMPTY 상태를 확인하고, fio 버전을 검증합니다.

QEMU null_blk fio 3.19+
3

Lab 1: 쓰기 제약 및 기본 성능 비교

ZNS(/dev/nullb0)와 Conventional SSD(/dev/nvme0n1)에 동일한 순차 쓰기를 수행해 처리량(Bandwidth)과 지연 시간을 비교합니다. 이어서 ZNS에 Random Write를 시도하여 Write Pointer 위반 에러(-EINVAL)를 직접 확인합니다.

fio (libaio) zonemode=zbd Sequential Write Random Write 위반
4

Lab 2: Zone 관리 상태 전이

blkzone report로 Zone 상태를 조회하고, Explicit Open → 쓰기 → Close → Finish → Reset 의 전체 상태 전이 시퀀스를 직접 실행하며 Write Pointer 이동을 관찰합니다.

blkzone report blkzone open/close/finish blkzone reset Write Pointer
5

Lab 3: WAF(쓰기 증폭 지수) 비교 측정

Pre-conditioning으로 Steady-State를 조성한 후 혼합 워크로드(랜덤 70% + 순차 30%)를 인가하고, /sys/block/DEV/stat 기반으로 WAF를 계산합니다. ZNS의 WAF ≈ 1과 Conventional SSD의 GC Copy-back 구조를 비교 분석합니다.

Pre-conditioning 혼합 워크로드 WAF = NAND / Host GC 구조 비교
6

Lab 4 (Case Study): ZNS 기술의 모바일 스토리지 확장 — Zoned UFS

Lab 1~3에서 직접 경험한 ZNS 제약들(쓰기 위반 에러, Zone 상태 전이, WAF≈1)이 모바일 스마트폰 환경(ZUFS)에서 어떤 시스템 과제로 이어지는지 탐구합니다. FAST '26 논문 Unleashing Zoned UFS를 통해 펌웨어~Android 5계층 크로스 레이어 최적화 전략을 학습합니다.

Step 4.1 스토리지 패러다임 변화 분석 (CUFS vs ZUFS, L2P 오버헤드 수백MB→수십KB) • Step 4.2 3대 과제 분석 (제한된 SRAM · 쓰기 순서 · GC 오버헤드) • Step 4.3 5계층 협력 아키텍처 (펌웨어~Android 크로스 레이어) • Step 4.4 실무 성능 평가 (단편화 환경 쓰기 2× 이상, 게임 로딩 14% 단축)
