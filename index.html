실습 개요

본 실습은 ZNS(Zoned Namespace) SSD의 핵심 특성인 쓰기 제약, Zone 상태 전이, WAF(Write Amplification Factor)를 직접 측정하고 Conventional SSD와 비교·분석합니다.

QEMU + Linux 커널 환경에서 null_blk 모듈로 NVMe SSD 역할의 /dev/nullb0와 ZNS SSD 역할의 /dev/nullb1 두 가상 장치를 구성합니다. fio·blkzone 명령으로 순차 쓰기 처리량 비교(Lab 1), Zone 상태 머신 조작(Lab 2), F2FS 파일시스템 레벨 WAF 측정(Lab 3)을 단계별로 실험합니다. 이어지는 Lab 4 (Case Study)에서는 동일한 Zone 개념이 모바일 스마트폰(Zoned UFS)에 이식될 때 발생하는 시스템 과제와 해결 전략을 FAST '26 논문을 통해 탐구합니다.

1

배경 지식: Zoned Storage Devices · Zone 동작 원리 · Zone 관리 명령 · Zone Append

LBA 공간의 Zone 분할과 순차 쓰기 제약·성능 이점(Zoned Storage Devices), Write Pointer 동작과 Zone 상태 전이(Zone 동작 원리), Reset·Open·Close·Finish 명령 의미(Zone 관리 명령), 원자적 LBA 할당과 병렬 I/O 지원(Zone Append)을 학습하고, Lab 1~3 실습과의 연결 관계를 확인합니다.

개념 학습 Write Pointer Zone 관리 명령 Zone Append 실습 연결
2

실습 환경 준비: null_blk 기반 가상 장치 2대 구성

null_blk 커널 모듈을 소스에서 빌드·적재하고, configfs를 이용해 NVMe SSD 역할의 /dev/nullb0(2 GB, 비-zone)과 ZNS SSD 역할의 /dev/nullb1(2 GB, Zone 크기 64 MB, 32 Zone)을 생성합니다. 각 장치의 Zone 인식과 EMPTY 상태를 확인하고, fio 버전을 검증합니다.

QEMU null_blk × 2 nullb0 NVMe / nullb1 ZNS fio 3.19+
3

Lab 1: 쓰기 제약 및 기본 성능 비교

NVMe SSD(/dev/nullb0)와 ZNS SSD(/dev/nullb1)에 동일한 순차 쓰기를 수행해 처리량(Bandwidth)과 지연 시간을 비교합니다. 이어서 ZNS에 Random Write를 시도하여 Write Pointer 위반 에러(-EINVAL)를 직접 확인합니다.

fio (libaio) zonemode=zbd Sequential Write Random Write 위반
4

Lab 2: Zone 관리 상태 전이

blkzone report로 Zone 상태를 조회하고, Explicit Open → 쓰기 → Close → Finish → Reset 의 전체 상태 전이 시퀀스를 직접 실행하며 Write Pointer 이동을 관찰합니다.

blkzone report blkzone open/close/finish blkzone reset Write Pointer
5

Lab 3: F2FS 파일시스템 레벨 WAF 측정

nullb1을 zone_nr_conv=1로 재구성하고 두 장치에 F2FS를 포맷·마운트합니다. GC 유발 워크로드(80% 채우기 + 랜덤 쓰기 60 s)를 수행한 뒤 /sys/block/nullb*/stat 기반으로 WAF를 계산합니다. NVMe F2FS의 GC Copy-back(WAF 1.2~2.0×)과 ZNS F2FS의 Zone-Reset 기반 GC(WAF ≈ 1.0×)를 비교 분석합니다.

Lab 4 (Case Study): ZNS 기술의 모바일 스토리지 확장 — Zoned UFS

Lab 1~3에서 직접 경험한 ZNS 제약들(쓰기 위반 에러, Zone 상태 전이, WAF≈1)이 모바일 스마트폰 환경(ZUFS)에서 어떤 시스템 과제로 이어지는지 탐구합니다. FAST '26 논문 Unleashing Zoned UFS를 통해 펌웨어~Android 5계층 크로스 레이어 최적화 전략을 학습합니다.
