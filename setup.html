실습 환경 준비

QEMU VM 실행 · null_blk ZNS 장치 구성 · 도구 설치

환경 전제 조건

이전 실습에서 구성한 QEMU 환경을 그대로 재사용합니다. 호스트에 아래 파일이 있어야 합니다.

ubuntu.qcow2 (30G) QEMU 5.x 이상 Linux 커널 5.18.0

장치 구성 목표

실습에서 사용할 세 가지 블록 장치의 역할을 미리 파악합니다.

/dev/vda
OS 디스크
virtio · ubuntu.qcow2
/dev/nullb0
Conventional SSD (비교 대상)
null_blk · zoned 없음 · 1GB
/dev/nullb1
ZNS SSD
null_blk · zoned=1 · 1GB
ℹ️ null_blk은 메모리 기반 에뮬레이터입니다. VM 재시작 시 데이터는 사라지지만 Zone 동작은 실제 ZNS SSD와 동일하게 재현됩니다.
단계별 설치 및 구성
1

QEMU VM 실행 (호스트에서)

OS 디스크만 연결합니다. 비교에 사용할 두 장치(nullb0, nullb1)는 VM 내부에서 null_blk으로 생성합니다.

bash — 호스트 
cd ~/qemu-lab

qemu-system-x86_64 \
  -m 4G -smp 4 \
  -enable-kvm \
  -drive file=ubuntu.qcow2,if=virtio,format=qcow2 \
  -net nic -net user,hostfwd=tcp::2222-:22 \
  -nographic

VM이 부팅되면 SSH로 접속합니다.

bash — 호스트에서 SSH 접속 
ssh -p 2222 user@localhost
⚠️ 이전 실습에서 사용했던 DPAS 커널(5.18.0-dpas)로 부팅해야합니다.
bash — VM 내부: DPAS 커널로 재부팅 
sudo grub-reboot "Advanced options for Ubuntu>Ubuntu, with Linux 5.18.0-dpas"
sudo reboot
2

필수 도구 설치 (VM 내부)

실습에 필요한 패키지를 설치합니다. blkzone은 util-linux에 포함되어 있습니다.

bash — VM 내부 
sudo apt-get update
sudo apt-get install -y \
  fio \           # 벤치마크 도구 (zonemode=zbd 내장)
  util-linux \    # blkzone 포함
  nvme-cli        # nvme list, smart-log 등

# 버전 확인
fio --version
blkzone --version
Expected output
fio-3.28
blkzone from util-linux 2.37.2
3

null_blk 커널 모듈 빌드 (VM 내부)

커널 5.18.0 소스 트리에서 null_blk을 모듈로 빌드합니다. 이미 빌드된 경우 이 단계를 건너뜁니다.

① menuconfig에서 모듈 선택

bash — VM 내부 
cd /linux-5.18.0
make menuconfig
# Device Drivers → Block devices
#   → Null test block driver  →  M (모듈) 선택 후 Save

② 빌드 및 설치

bash — VM 내부 
# 블록 드라이버 디렉토리만 빌드 (전체 빌드 불필요)
make M=drivers/block -j$(nproc)
sudo make M=drivers/block modules_install

# 모듈 확인
modinfo null_blk | grep filename
4

null_blk 장치 생성 — setup_nullb.sh (VM 내부)

아래 스크립트를 저장한 뒤 실행합니다. 내부적으로 nr_devices=0으로 모듈을 올린 뒤 configfs로 Conventional SSD(nullb0)와 ZNS SSD(nullb1)를 순서대로 생성합니다. power 순서가 장치 번호를 결정하므로 nvme0을 먼저 활성화합니다. 두 장치의 유일한 차이는 zoned 속성 유무이며, VM 재시작 후에도 이 스크립트를 다시 실행하면 동일하게 복구됩니다.

bash — setup_nullb.sh (저장 후 재사용) 
#!/usr/bin/env bash
# setup_nullb.sh — null_blk 장치 초기화 (Conventional SSD + ZNS SSD)
# 사용법: sudo bash setup_nullb.sh

set -euo pipefail

sudo modprobe null_blk nr_devices=0

## nullb0: Conventional SSD (zoned 없음) — 먼저 power=1
sudo mkdir /sys/kernel/config/nullb/nvme0
sudo sh -c 'echo 4096 > /sys/kernel/config/nullb/nvme0/blocksize'
sudo sh -c 'echo 1024 > /sys/kernel/config/nullb/nvme0/size'
sudo sh -c 'echo 1    > /sys/kernel/config/nullb/nvme0/memory_backed'
sudo sh -c 'echo 1    > /sys/kernel/config/nullb/nvme0/power'

## nullb1: ZNS SSD (zoned=1) — 두 번째 power=1
sudo mkdir /sys/kernel/config/nullb/zns0
sudo sh -c 'echo 1    > /sys/kernel/config/nullb/zns0/zoned'
sudo sh -c 'echo 64   > /sys/kernel/config/nullb/zns0/zone_size'
sudo sh -c 'echo 16   > /sys/kernel/config/nullb/zns0/zone_max_open'
sudo sh -c 'echo 16   > /sys/kernel/config/nullb/zns0/zone_max_active'
sudo sh -c 'echo 4096 > /sys/kernel/config/nullb/zns0/blocksize'
sudo sh -c 'echo 1024 > /sys/kernel/config/nullb/zns0/size'
sudo sh -c 'echo 1    > /sys/kernel/config/nullb/zns0/memory_backed'
sudo sh -c 'echo 1    > /sys/kernel/config/nullb/zns0/power'

lsblk /dev/nullb0 /dev/nullb1
printf '\n✔ nullb0 (Conventional SSD) · nullb1 (ZNS SSD) 준비 완료\n'
bash — 실행 
sudo bash setup_nullb.sh
Expected output
NAME      MAJ:MIN RM SIZE RO TYPE MOUNTPOINTS
nullb0    251:0    0   1G  0 disk   ← Conventional SSD
nullb1    251:1    0   1G  0 disk   ← ZNS SSD
⚠️ null_blk 장치는 메모리 기반이라 VM 재시작 시 사라집니다. 재부팅 후에는 스크립트를 다시 실행하세요.

ZNS 장치 동작 확인

blkzone report로 Zone 목록을 조회하여 ZNS 장치가 정상 구성됐는지 확인합니다.

bash — VM 내부 
sudo blkzone report /dev/nullb1 | head -5
Expected output
  start: 0x000000000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 1(em) [type: 2(SEQ_WRITE_REQUIRED)]
  start: 0x000020000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 1(em) [type: 2(SEQ_WRITE_REQUIRED)]
  start: 0x000040000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 1(em) [type: 2(SEQ_WRITE_REQUIRED)]
  start: 0x000060000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 1(em) [type: 2(SEQ_WRITE_REQUIRED)]
  start: 0x000080000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 1(em) [type: 2(SEQ_WRITE_REQUIRED)]
✅ 모든 Zone이 zcond: 1(em)이면 ZNS 장치가 정상입니다.
